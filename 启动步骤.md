# Campus-Gym-Pro 启动步骤

## 一、本地开发环境启动

### 1. 前置依赖

| 依赖 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 17 或 21（推荐 21） | **不能使用 JDK 21 以上版本**，否则 Lombok 注解处理器不工作 |
| Maven | 3.8+ | 构建工具 |
| Node.js | 18+ | 前端开发服务器 |
| MySQL | 8.0+ | 数据库 |
| Redis | 7.0+ | 缓存 / 分布式锁 |
| RabbitMQ | 3.12+ | 消息队列（预约异步处理） |

### 2. 启动基础设施

确保 MySQL、Redis、RabbitMQ 已启动。可以用 Docker 只启动这三个服务：

```bash
# 在项目根目录执行（会自动跳过未运行的 backend/frontend）
docker compose up -d mysql redis rabbitmq
```

> **注意**：如果本地已有 MySQL 占用 3306 端口，Docker 容器会启动失败。
> 可以直接使用本地 MySQL，无需启动 Docker 的 MySQL 容器。

### 3. 初始化数据库

```bash
mysql -u root -p116688hsj -e "CREATE DATABASE IF NOT EXISTS campus_gym DEFAULT CHARACTER SET utf8mb4;"
mysql -u root -p116688hsj campus_gym < sql/init.sql
```

根据实际情况替换用户名和密码。数据库连接配置在 `src/main/resources/application.yml`。

### 4. 编译并启动后端

```bash
# 必须指定 JAVA_HOME 为 JDK 17 或 21
JAVA_HOME=/Users/GXA/Library/Java/JavaVirtualMachines/temurin-21/Contents/Home \
  mvn spring-boot:run -DskipTests -B
```

> **常见错误：Lombok 注解处理器失效**
>
> 症状：编译报错 `找不到符号 方法 getUserId() / getStatus() / log`，或运行时报
> `NoSuchMethodError: 'java.lang.Integer com.campusgym.pro.entity.Slot.getStatus()'`
>
> 原因：`brew install maven` 会同时安装 openjdk（可能是 26+），Maven 默认使用该版本。
> Lombok 1.18.32 不支持 JDK 22+，导致 `@Data`、`@Slf4j` 等注解不生效。
>
> 解决方案：编译和运行时指定 JAVA_HOME 为 JDK 17 或 21：
> ```bash
> JAVA_HOME=/path/to/jdk-21 mvn spring-boot:run -DskipTests -B
> ```

后端启动成功标志：`Started CampusGymProApplication in X seconds`

### 5. 启动前端

```bash
cd frontend
npm install   # 首次需要安装依赖
npm run dev
```

前端启动后访问 http://localhost:3000

### 6. 验证

```bash
# 后端 API
curl http://localhost:8080/api/stadiums

# 前端页面
curl -o /dev/null -w "%{http_code}" http://localhost:3000/
```

## 二、Docker 全量启动（新主机）

### 1. 前置条件

- 安装 [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- 确保 Docker 运行正常：`docker info`

### 2. 一键启动

```bash
git clone <仓库地址>
cd Campus-Gym-Pro
docker compose up -d --build
```

首次启动会拉取镜像并构建，耗时较长。后续启动直接 `docker compose up -d` 即可。

### 3. 服务列表

| 服务 | 端口 | 说明 |
|------|------|------|
| 前端 (nginx) | 80 | http://localhost |
| 后端 (Spring Boot) | 8080 | http://localhost:8080 |
| MySQL | 3306 | 用户 root，密码 root，数据库 campus_gym |
| Redis | 6379 | 无密码 |
| RabbitMQ | 5672 / 15672 | 管理界面 http://localhost:15672（guest/guest） |

### 4. 常见问题

**端口被占用**

```bash
# 查看占用端口的进程
lsof -i :3306
# 停止本地 MySQL
brew services stop mysql
```

**镜像拉取失败（国内网络）**

在 `docker-compose.yml` 中添加镜像源，或配置 Docker Daemon 的 `registry-mirrors`：

```json
{
  "registry-mirrors": ["https://docker.1ms.run"]
}
```

**Apple Silicon (M1/M2/M3) 平台兼容问题**

如果构建报 `no match for platform in manifest`，在 Dockerfile 的 FROM 行指定平台：

```dockerfile
FROM --platform=linux/amd64 maven:3.9-eclipse-temurin-17-alpine AS build
```

或在 docker-compose.yml 中添加：

```yaml
services:
  backend:
    platform: linux/amd64
```

### 5. 常用命令

```bash
# 查看日志
docker compose logs -f backend
docker compose logs -f frontend

# 重启单个服务
docker compose restart backend

# 停止所有服务
docker compose down

# 停止并删除数据卷（清空数据库）
docker compose down -v
```

## 三、错误排查速查表

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| `NoSuchMethodError: getStatus()` / `getUserId()` | Lombok 注解处理器因 JDK 版本过高未生效 | 指定 `JAVA_HOME` 为 JDK 17 或 21 重新编译 |
| `找不到符号: 方法 log` | `@Slf4j` 注解未处理 | 同上，降低 JDK 版本 |
| `ports are not available: 3306` | 本地 MySQL 已占用端口 | 停止本地 MySQL 或在 docker-compose.yml 中修改端口映射 |
| `no match for platform in manifest` | Apple Silicon 拉取 x86 镜像失败 | 添加 `--platform=linux/amd64` 或使用多平台镜像 |
| 前端显示"无法连接到服务器" | 后端未启动或端口未通 | 先确保后端运行在 8080 端口 |
| `maven:3.9-eclipse-temurin-17-alpine` 拉取失败 | Docker 镜像源限流 | 更换镜像源或本地安装 Maven + JDK |
